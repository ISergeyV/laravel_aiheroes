# 🚀 Идеальная шпаргалка: Развертывание Laravel на Хостинге

Эта шпаргалка содержит **полную последовательность команд от А до Я** для развертывания вашего проекта на хостинге (через SSH).

Просто копируйте и вставляйте их по порядку.

---

### Шаг 1: Подготовка папки и загрузка кода
*Важно: убедитесь, что вы находитесь в правильной папке (например, `cd /home/isergeyv/aiheroes.net/www/`)*

1. Полностью очищаем папку (включая все скрытые файлы):
```bash
rm -rf .[^.]* *
```

2. Клонируем проект из GitHub текущую пустую папку (обратите внимание на точку в конце!):
```bash
git clone https://github.com/ISergeyV/laravel_aiheroes.git .
```

---

### Шаг 2: Обновление и Установка зависимостей (Composer)
**Важное правило:** Мы обновляем версии пакетов ТОЛЬКО локально (`composer update`), а на сервере (хостинге) только устанавливаем точные версии (`composer install`), чтобы сайт не сломался от неожиданных обновлений.

> ⚠️ ИЛИ как работать с PHP на виртуальном хостинге:
> На виртуальных хостингах терминальная команда `php` (и `composer`) по умолчанию часто запускает старую версию PHP (например, 8.3). Для Laravel 11+ или новых пакетов вам нужно использовать команду для PHP 8.4 напрямую.
> 
> **Вместо стандартных команд делайте так:**
> ```bash
> # Проверяем, какой ярлык отвечает за PHP 8.4:
> php8.4 -v 
> # или
> php84 -v
> 
> # Создаем удобный алиас (работает до закрытия терминала):
> alias php='php8.4'
> 
> # Устанавливаем зависимости, явно указывая правильный PHP и путь до composer:
> php8.4 /usr/local/bin/composer install --optimize-autoloader --no-dev
> ```

### Шаг 3: Настройка окружения (.env)
1. Создаем рабочий файл с настройками:
```bash
cp .env.example .env
```

2. Генерируем уникальный ключ безопасности (он сам запишется в `.env`):
```bash
php artisan key:generate
```

3. **Откройте файл `.env` (например, через `nano .env`) и пропишите доступы!**
   *   `APP_ENV=production`
   *   `APP_DEBUG=false`
   *   `DB_HOST=127.0.0.1` *(или другой, который выдал хостинг)*
   *   `DB_DATABASE=имя_вашей_базы`
   *   `DB_USERNAME=имя_пользователя_базы`
   *   `DB_PASSWORD="ваш_пароль_в_кавычках"`
   *   `WEBHOOK_SECRET="ваш_секретный_токен_для_вебхука"`
   
   *(💡 Важно: После любого изменения файла `.env`, обязательно выполните `php artisan config:clear`, чтобы сайт увидел новые настройки, особенно это касается вебхука!)*

---

### Шаг 4: База данных и Пользователи
*Эти команды выполняются строго ПОСЛЕ того, как вы сохранили правильный файл `.env`*

1. Сбрасываем кэш старых настроек (на всякий случай):
```bash
php artisan config:clear
php artisan cache:clear
```

2. Создаем все таблицы в базе данных:
```bash
php artisan migrate --force
```

3. Загружаем стартовые данные в базу (настройки сайта, базовые тексты, если у вас есть сидеры):
```bash
php artisan db:seed --force
```

4. **Создаем Администратора для входа в панель Filament (то, что вы просили!):**
```bash
php artisan make:filament-user
```
*(Система в терминале по очереди спросит: Name, Email и Password. Введите их, и пользователь будет создан)*

---

### Шаг 5: Изображения и Оптимизация (Финальный штрих)

1. Делаем картинки из админки `storage` доступными публично в интернете:
```bash
php artisan storage:link
```

2. Запускаем "турбо-режим" Laravel (кэшируем файлы конфигурации, маршруты и интерфейс для максимальной скорости):
```bash
php artisan optimize
php artisan view:cache
```
